### QUA-28 ‚Äî qualitool-core: Parallel executor with max_parallelism
- **Status**: Complete
- **Branch**: feat/QUA-28-parallel-executor (based on feat/QUA-27-dag-scheduler)
- **Project**: Architecture bootstrap ‚Äî v1 skeleton
- **What was done**:
  - Implemented `SchedulerConfig` with `max_parallelism` field (default: `available_parallelism - 1`, minimum 1)
  - Implemented `Scheduler::run_parallel()` ‚Äî bounded-concurrency executor using wave-front algorithm
  - Wave-front algorithm: in-degree tracking, reverse-dep map, ready queue dispatch
  - Concurrency bounded by `tokio::sync::Semaphore` to honor `max_parallelism`
  - Spawned tasks via `tokio::task::JoinSet` ‚Äî dropping the future cancels in-flight tasks (graceful shutdown)
  - Probe outputs shared via `Arc<tokio::sync::Mutex<HashMap>>` for safe concurrent reads
  - Internal storage changed from `Box<dyn DynProbe/DynCheck>` to `Arc<dyn DynProbe/DynCheck>` for `'static` spawning
  - Preserved existing sequential `Scheduler::run()` path unchanged
  - All exit criteria verified passing:
    - `cargo check --workspace` ‚Äî passes
    - `cargo test -p qualitool-core` ‚Äî 48 tests pass (37 existing + 11 parallel executor tests)
