### QUA-27 — qualitool-core: DAG scheduler with topological sort & cycle detection
- **Status**: Complete
- **Branch**: feat/QUA-27-dag-scheduler (based on feat/QUA-26-core-traits)
- **Project**: Architecture bootstrap — v1 skeleton
- **What was done**:
  - Implemented `NodeId` enum (Probe/Check) with derived `Ord` for deterministic ordering (probes before checks)
  - Implemented `ScheduleError` with `Cycle`, `UnknownDependency`, `DuplicateNode` variants
  - Implemented `RunError` with `ProbeFailed`, `CheckFailed` variants wrapping source errors
  - Implemented `SchedulerBuilder` with fluent `add_probe`/`add_check` API and `build()` validation

  - Cycle detection reports the full cycle path (e.g., `probe:A -> probe:B -> probe:C -> probe:A`)
  - Implemented `Scheduler::run()` — sequential executor with probe output memoization
  - Probe outputs collected during execution are injected into downstream ProbeContext/CheckContext
  - Type-erased Probe/Check trait objects via internal DynProbe/DynCheck traits (boxes RPITIT futures)
