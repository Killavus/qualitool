- **What was done**:
  - Implemented `SchedulerConfig` with `max_parallelism` field (default: `available_parallelism - 1`, minimum 1)
  - Implemented `Scheduler::run_parallel()` тАФ bounded-concurrency executor using wave-front algorithm
  - Wave-front algorithm: in-degree tracking, reverse-dep map, ready queue dispatch
  - Concurrency bounded by `tokio::sync::Semaphore` to honor `max_parallelism`

  - Preserved existing sequential `Scheduler::run()` path unchanged
